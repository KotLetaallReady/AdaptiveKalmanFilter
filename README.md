# AdaptiveKalmanFilter

Android-приложение для демонстрации адаптивной фильтрации GPS-траектории с помощью фильтра Калмана и нейросетевого пост-сглаживания.

Проект состоит из двух частей:

* `core:*` — библиотечные модули с моделями, фильтром, метриками, нейросетевым сглаживателем и Android-адаптерами.
* `app` — демонстрационное приложение для диплома: запуск фильтра на реальных данных телефона, обучение сглаживателя, сравнение результатов и экспорт CSV.

## Возможности

* получение GPS-измерений и IMU-данных с Android-устройства;
* классический фильтр Калмана для двумерной траектории;
* адаптация шума измерений `R` по точности GPS и статистике инноваций;
* обучение компактной MLP-модели для выбора коэффициента сглаживания `alpha`;
* нейросетевое пост-сглаживание траектории поверх результата Калмана;
* сравнение raw GPS, Kalman и neural smoother в одном режиме;
* расчет метрик качества: RMSE, MAE, максимальная ошибка, стабильность, лаг;
* экспорт сравнения в CSV для дальнейшего анализа и графиков.

## Режимы приложения

### Классический фильтр

Запускает GPS-сессию и строит траекторию после обработки фильтром Калмана. На экране отображаются координаты, скорость, матрица шума измерений `R`, коэффициенты Калмана `K`, инновация и оценка неопределенности позиции.

### Нейросетевой фильтр

Работает в три этапа:

1. Сбор данных на реальном маршруте через классический фильтр.
2. Обучение MLP-модели на признаках окна траектории.
3. Повторный запуск маршрута с пост-сглаживанием результата Калмана.

Обученная модель сохраняется в приватные файлы приложения и может использоваться в следующих запусках.

### Сравнение фильтров

Параллельно показывает raw GPS, результат Калмана и результат нейросетевого сглаживателя. После остановки сессии приложение считает метрики и сохраняет CSV-файл в Downloads.

## Архитектура

```text
app
  Демонстрационное Android-приложение на Jetpack Compose.

core:model
  Общие модели данных: Observation, AccuracyMetrics, ComparisonRow.

core:data
  Абстракция источника сенсорных данных: SensorDataSource.

core:engine
  Чистый Kotlin/JVM-движок: KalmanFilter, стратегии коэффициентов,
  метрики, нейросеть, обучение и fixed-lag smoother.

core:android
  Android-реализации интерфейсов движка: GPS/IMU source, Logcat logger,
  файловое хранилище обученной модели.
```

Библиотечная часть намеренно отделена от UI. Модули `core:model`, `core:data` и `core:engine` не зависят от Android и могут тестироваться как обычный JVM-код. Android-зависимости вынесены в `core:android`.

## Основные классы библиотеки

* `Observation` — одно GPS/IMU-измерение.
* `SensorDataSource` — интерфейс источника измерений.
* `AndroidSensorDataSource` — Android-источник GPS и IMU.
* `KalmanFilter` — фильтр Калмана с локальной системой координат.
* `ClassicalCoefficientStrategy` — классический расчет `K`, `R` и posterior covariance.
* `KalmanFilterCoordinator` — высокоуровневый runner с `Flow<FilterResult>`.
* `MetricsEvaluator` — расчет метрик качества траектории.
* `NeuralTrajectorySmoother` — fixed-lag сглаживание результата Калмана.
* `SmoothingTrainingCollector` — сбор обучающего датасета для smoother-модели.
* `NeuralNetworkTrainer` — обучение MLP-модели.
* `SmootherRepository` и `SmootherStore` — сохранение и загрузка обученной модели.

## Требования

* Android Studio с поддержкой Kotlin и Gradle;
* JDK 11 или новее;
* Android SDK 36;
* устройство или эмулятор с Android 7.0+ (`minSdk = 24`);
* для реального тестирования желательно физическое устройство с GPS и датчиками движения.

## Запуск

1. Открыть проект в Android Studio.
2. Дождаться Gradle Sync.
3. Подключить Android-устройство или запустить эмулятор.
4. Запустить конфигурацию `app`.
5. При первом запуске выдать разрешение на геолокацию.

Для запуска тестов:

```bash
./gradlew test
```

Для сборки debug APK:

```bash
./gradlew assembleDebug
```

На Windows можно использовать `gradlew.bat`:

```bash
gradlew.bat test
gradlew.bat assembleDebug
```

## Разрешения

Приложение использует:

* `ACCESS_FINE_LOCATION` и `ACCESS_COARSE_LOCATION` для GPS;
* `WAKE_LOCK`, чтобы устройство не останавливало GPS-сессию во время записи;
* `WRITE_EXTERNAL_STORAGE` только на Android 9 и ниже для экспорта CSV.

На Android 10+ CSV сохраняется через `MediaStore`, поэтому отдельное разрешение на запись во внешнее хранилище не требуется.

## CSV-экспорт

В режиме сравнения приложение сохраняет таблицу с такими данными:

* raw GPS координаты;
* GPS accuracy и speed;
* координаты фильтра Калмана;
* скорость и неопределенность Калмана;
* Savitzky-Golay оценка;
* итоговая smoothed траектория;
* коэффициент `alpha`, предсказанный нейросетью.

Файл создается в Downloads с именем вида:

```text
kalman_comparison_yyyyMMdd_HHmmss.csv
```

## Отделение библиотеки от приложения

Если проект публикуется как два отдельных репозитория, разумное разделение такое:

* библиотека: `core:model`, `core:data`, `core:engine`, при необходимости `core:android`;
* демо-приложение: `app`, UI, навигация, Hilt DI, CSV-экспорт и дипломные сценарии;
* исследовательские скрипты и готовые изображения: оставить вне библиотечного репозитория или вынести в отдельную папку дипломных материалов.

Так библиотека остается компактной и переиспользуемой, а приложение показывает практический сценарий ее работы.

## Статус проекта

Проект разработан как дипломная демонстрация алгоритма адаптивной фильтрации GPS-траектории. Код `core` оформлен как основа для отдельной библиотеки, а `app` служит визуальной демонстрацией, инструментом сбора данных и способом сравнить классический и нейросетевой варианты обработки.
